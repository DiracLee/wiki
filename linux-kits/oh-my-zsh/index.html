    # zsh 简介

    Z Shell，简称 ZSH，是由 Paul Falstad 于 1990 年开发的 Unix 命令行解释器。我们可以将 ZSH 视为 Bourne Again Shell (BASH) 的扩展版本，因为它具有类似于 bash 的功能。

    多年来，ZSH 因其特性、灵活性和可定制性而成为 Unix 系统的流行 shell 之一。值得注意的 ZSH 功能包括：

    1. 自动指令修正
    2. 命令自动完成
    3. 改进的 shell 变量处理
    4. 易于设置和外壳定制

    # oh-my-zsh 简介

    Oh-my-zsh 是 ZSH shell 的免费开源框架。Oh-my-zsh 用于帮助有效管理和配置 ZSH shell。它包含用于改进默认 ZSH shell 的功能、主题和插件。

    # 安装和配置 zsh

    安装 ZSH shell 是安装 oh-my-zsh 的前提，默认情况下，大多数 Linux 发行版中都没有预安装 ZSH，您需要手动安装它。

    您可以使用系统的默认包管理器来安装它。

    sudo apt update
sudo apt install zsh

    确保使用 which 命令确保安装了 ZSH：

    which zsh

    您应该会看到

    /usr/bin/zsh

    更改默认 shell

    chsh -s $(which zsh)

    重新登陆当前账户时，将会看到 zsh 提示您尚未配置 zsh

    选择 0，创建一个空的 .zshrc 配置文件。

    # 安装和配置 oh-my-zsh

    安装 oh-my-zsh 框架的最简单方法是使用提供的安装程序脚本。

    我们需要确保在下载脚本之前安装了 curl 和 git。

    sudo apt-get install git curl

    然后使用以下命令下载并安装 oh-my-zsh 安装脚本：

    sh -c "$(curl -fsSL https://raw.github.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

    # 配置插件

    在 ~/.zshrc 中更改 plugins 的值就能配置插件，例如：

    plugins=(git)

    # 自带插件推荐

    # 1. git

    定义了有关 git 的 alias。常用的有

    gaa = git add --all
gcmsg = git commit -m
ga = git add
gst = git status
gp = git push

    # 2.tmux

    定义了有关 tmux 的 alias。常用的有

    tl = tmux list-sessions
tkss = tmux kill-session -t
ta = tmux attach -t
ts = tmux new-session -s

    # 3. extract

    提供一个 extract 命令，以及它的别名 x。功能是：一！键！解！压！

    你知道tar的四种写法吗？我也不知道，所以我装了这个。从今以后 tar, gz, zip, rar 全部使用 extract 命令解压，再也不用天天查 cheatsheet 啦！

    # 4. themes

    提供一条 theme 命令，用来随时手动切换主题。在想要尝试各种主题的时候很实用，不需要一直改 zshrc 然后重载。

    # 5. gitignore

    提供一条 gi 命令，用来查询 gitignore 模板。比如你新建了一个 python 项目，就可以用

    gi python > .gitignore

    来生成一份 gitignore 文件。

    # 6. cp

    提供一个 cpv 命令，这个命令使用 rsync 实现带进度条的复制功能。

    # 7. z

    提供一个 z 命令，在常用目录之间跳转。类似 autojump，但是不需要额外安装软件。

    # 8. colored-man-pages

    给你带颜色的 man 命令。

    # 9. sudo

    apt 忘了加 sudo？开启这个插件，双击 Esc，zsh 会把上一条命令加上 sudo 给你。

    # 第三方插件推荐

    # 1. zsh-syntax-highlighting

    克隆到 oh-my-zsh 主题目录

    git clone https://github.com/zsh-users/zsh-syntax-highlighting.git ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-syntax-highlighting

    在 ~/.zshrc 中配置此插件

    plugins=( [plugins...] zsh-syntax-highlighting)

    shell 命令的代码高亮。你没有理由拒绝高亮。

    # 2. zsh-autosuggestions

    在输入命令的过程中根据你的历史记录显示你可能想要输入的命令，按 tab 补全。
